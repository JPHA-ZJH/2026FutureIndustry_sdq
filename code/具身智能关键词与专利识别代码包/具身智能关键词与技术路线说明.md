# 具身智能关键词、技术路线与专利识别说明

## 一、结论与分类口径

可以从两份用户提供的报告中形成较稳定的具身智能技术分类，并据此构建中文专利关键词词典。词典以中文术语为主体，仅保留中文材料和中文专利中常见的英文缩写或模型名，例如 VLA、VLM、LLM、MPC、WBC、ROS、AMR、eVTOL、RGB-D、SLAM、IMU、QDD、PPO 等；不另行复制一套完整英文词典。

两份报告的功能不同：

- 《具身智能发展报告（2025年）》适合确定技术框架、技术路线、产品形态、数据与产业生态。报告明确提出“数据—模型—本体”三个关键要素和云边端协同框架，并将模型技术路径分为模块化分层、分层大模型、端到端大模型和世界模型四条路线（报告标注页码第8—29页，技术路径图见第14页）。
- 《2025中国具身智能产业星图》更适合补充产业化词汇，尤其是端侧芯片、存储、通信模组、数据力、电力，以及灵巧手、末端执行器、减速器、驱动器、丝杠、电机、一体化关节、轴承、传感器等底座和核心部件。它是产业生态图，不宜单独作为算法技术分类。

在此基础上，又采用可公开核验的科研机构和资本市场报告补充前沿与工程化词汇。科研侧重点补充具身感知、具身操作、空间推理、持续学习、群体具身智能、真实世界评测、模型加速、安全与价值对齐等方向；资本市场报告重点补充扩散策略、ACT、IMU、MEMS触觉、旋转/线性执行器、QDD/SEA、编码器和丝杠细分等工程词汇。

## 三、专利识别与评分规则

### 1. 先判断语境，再判断技术核心度

- “具身智能、具身基础模型、VLA”等专用词可以独立触发。
- 人形机器人、四足机器人、轮臂机器人、灵巧手等典型物理载体可以独立触发。
- 世界模型、强化学习、扩散策略、触觉传感器、电机、轴承等通用词，必须与机器人或其他物理本体语境共现后才生效。
- 无人机、无人车等交叉产业载体还必须出现自主感知、决策、导航、学习或控制等智能语境。
- 聊天机器人、客服机器人、RPA、数字人等设为负向消歧词；只有同时出现明确物理本体或明确具身专用词时才保守保留。

### 2. 最终分为1—5分，不输出离散相关性标签

| 分值 | 含义 | 典型技术 |
|---:|---|---|
| 5 | 具身智能核心范式或核心模型路线 | 具身智能、具身基础模型、VLA、世界模型、智能闭环 |
| 4 | 关键算法、学习、感知、操作或高关键度部件 | 强化学习、MPC/WBC、主动感知、灵巧手、六维力矩/触觉传感器 |
| 3 | 重要本体、平台、执行器、数据与部署能力 | AMR、仿真合成、云边端、QDD、线性/旋转执行器 |
| 2 | 相关支撑技术或交叉载体 | 减速器、丝杠、惯性传感、智能化无人机载体 |
| 1 | 高通用度外围部件、材料、能源或应用词 | 电机、轴承、编码器、固态电池、家庭/商业应用 |

最终 `embodied_score` 采用实际技术类型的最高核心度，而不简单累加关键词。例如：

- “机器人利用世界模型预测未来状态并规划动作”识别为世界模型路线，5分。
- “人形机器人关节用精密轴承”通过人形机器人确认语境，但按轴承这一实际技术类型记1分。
- “一种轴承加工方法”没有具身或机器人本体语境，不进入样本。
- “聊天机器人世界模型”命中纯软件机器人负向规则，归零。
- “无人机自主导航与避障”满足载体与智能语境，按自主任务能力记4分。

`embodied_score_raw` 保留所有有效规则的证据累计分，便于排序、审查和阈值实验；它不代表技术核心程度。用户若希望保留所有已通过语境校验的外围专利，可令 `MIN_EMBODIED_SCORE = 1`；希望更严格时可改为2或3。

## 五、代码文件关系

- `embodied_matcher_v1_optimized_region_newsource.py`：关键词规则、上下文消歧、1—5分评分、技术领域/细分方向分类、企业汇总和词典导出。
- `embodied_firm_2021_2025_newsource.py`：按照原量子脚本的数据口径，逐年分块读取2021—2025年数据，仅保留发明授权，提取第一申请人及地区/城市，调用匹配器并输出专利、企业—年份、企业和关键词词典四类CSV。
