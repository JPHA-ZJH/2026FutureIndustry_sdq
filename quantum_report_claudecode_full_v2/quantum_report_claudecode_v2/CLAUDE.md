# Claude Code 项目规则：上海量子信息产业咨询报告

## 一、固定工作顺序

1. 先运行 `python run_all.py`。
2. 阅读 `outputs/audit/data_cleaning_audit.md` 和四个审计CSV。
3. 检查 `outputs/tables` 与 `outputs/figures` 是否完整、数字是否一致。
4. 再联网检索第一节技术路线和第六节企业基本情况所需的权威资料。
5. 严格按照 `docs/report_outline.md` 写初稿。
6. 完成数字核验和外部来源核验后，才能润色。

不得跳过数据清洗和结果核验直接写报告。

## 二、不可擅自改变的清洗规则

- “上海市”和“上海”必须统一为“上海”，其他城市同理。
- 城市到省份的映射使用 `data/reference` 中的全国行政区参考表；未知城市进入审计清单，不得主观猜测。
- 合作专利的原始“申请人类型”可能包含所有申请人的类型，不能直接作为第一申请人类型。
- 必须重新提取第一申请人，并依据第一申请人名称形成唯一的 `主体类型`。
- 企业分析仅使用 `主体类型=企业`；高校和科研院所单独分析；待复核主体不进入企业排名。
- 如需修改类型识别或城市映射规则，必须先说明影响并重新运行全部代码。

## 三、固定分析边界

- 技术内部分类只使用五个方向：量子计算、量子通信与安全、量子传感、量子计算硬件、量子基础概念。
- 不分析高相关、中相关、核心口径等相关性分类。
- 不分析产业链环节，不引用 `chain_position` 形成结论。
- 被引字段覆盖率不足，不把平均被引或高被引作为主指标。
- 专利数据不能证明产值、收入、市场份额、融资、产品销量、产业化程度或技术先进性。
- 最新年份可能存在授权滞后和数据更新不完整问题，趋势判断必须保守。

## 四、固定章节

报告只能使用以下七节，不得自行增加或合并编号章节：

1. 全国量子科技技术路线及专利演进
2. 上海量子信息创新发展的总体特征
3. 上海量子信息企业主体与发展梯队
4. 上海量子信息企业技术布局
5. 上海高校和科研院所的技术支撑
6. 上海及其他地区代表性量子信息企业
7. 主要问题与政策建议

各节具体小节和写作要求见 `docs/report_outline.md`。

## 五、外部资料规则

- 第一节技术路线优先使用国家发展改革委、中国信通院、中国科学院、国家自然科学基金委等官方或权威科研机构资料。
- 第六节企业资料优先使用企业官网、政府网站、上市公司公告和正式年报。
- 商业数据库、招聘网站和普通媒体只能作为线索，不得作为关键事实的唯一依据。
- 所有外部事实记录到 `report/source_log.csv`，包括链接、发布日期、支持的具体表述和来源等级。

## 六、图表规则

- 优先使用 `outputs/figures` 中已经生成的图。
- 正文每张图都必须有图号、标题、数据口径和1—2句解释。
- 图表用于支持结论，正文不要机械重复全部数字。
- 不得为了“丰富”报告而添加无法从数据或权威资料得到的图表。
