# 版本说明：生物制造 Claude Code 工作包 V2

## 相对量子信息工作包的主要适配

### 1. 技术分类改为生物制造专属口径

原始专利识别结果包含13类技术方向，分析底表完整保留；为提高政府咨询报告的可读性，正文归并为7条技术路线：

1. 生物设计、读写与自动化工具；
2. 酶与蛋白质工程；
3. 细胞工厂与菌株工程；
4. 生物过程与规模化；
5. 原料与低碳路线；
6. 生物制造产品与应用；
7. 前沿制造与基础支撑。

归并关系记录在 `config.py` 和 `outputs/audit/technology_route_mapping_audit.csv`。

### 2. 主体类型清洗进一步细化

除企业、高校和科研院所外，生物制造数据中医疗机构较多，因此新增“医疗机构”独立类型，避免医院被误归为科研院所或高校。

合作专利原始类型可能同时包含“企业,学校”等多个类型。新版不直接沿用混合类型，而是提取第一申请人并依据名称规则形成唯一主体类型；无法可靠判断的记录进入人工复核清单。

### 3. 城市标准化面向全国数据

统一“上海/上海市”等写法，结合全国城市参考表映射省份；后续出现当前样本之外的中国城市时，能够自动识别或进入未映射清单，而不是只维护现有城市列表。

### 4. 调整报告分析范围

- 不分析专利相关性等级；
- 不分析产业链环节；
- 技术内部比较以7条报告技术路线为主；
- 被引证次数覆盖率较低，不作为主要质量指标；
- 以企业为主，高校和科研院所作为科研支撑补充。

### 5. 增加地区主体结构比较

第五节第（四）部分新增：

- 图14：重点省市企业、高校和科研院所专利占比；
- 图15：重点省市高校和科研院所专利绝对数量。

图14以三类主体专利合计为分母，医疗机构、个人、机关团体和其他机构不进入分母。图15用于避免只比较占比而忽视绝对规模。

### 6. 初稿写作约束进一步强化

Claude Code 必须严格按照七节提纲撰写，不得自行增删一级节。每项数字必须能够回溯到分析底表；技术路线和企业情况需通过政府部门、权威科研机构、企业官网或正式公告核验；无法支持的判断必须删除或降格表述。
