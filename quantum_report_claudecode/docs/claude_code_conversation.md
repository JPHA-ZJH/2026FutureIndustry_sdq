# 建议在 Claude Code 中进行的完整对话

## 第1轮：数据审计与代码运行

把 `prompts/01_run_and_audit.md` 全文发送给 Claude Code。

验收时你可以继续问：

> 你生成的审计结果里，哪些问题会导致报告结论失真？请只列必须先处理的问题，并直接修改代码重新运行。

然后问：

> 请打开所有生成图形，检查中文、排序、图例、单位和数据标签。只保留对政府报告有解释价值的图，不要为凑数量画图。

## 第2轮：确认分析口径

发送：

> 在写报告前，请依据 `outputs/analysis_summary.md` 给出不超过15条“可以写入正文的客观发现”。每条都注明对应表格文件。不得写政策建议，不得补充网络资料。

检查这些发现是否准确。发现错误时要求其返回代码和表格修正，而不是改写文字掩盖问题。

## 第3轮：撰写初稿

把 `prompts/02_write_draft.md` 全文发送。

如果 Claude Code 不能联网，补充：

> 第一节和企业公开信息暂时保留“待补充权威来源”标记，不得用模型记忆填充。数据分析章节先完整写出。

## 第4轮：企业案例校准

发送：

> 请说明上海3家和其他省市3—5家企业的入选规则，并分别列出：专利数据事实、需要联网核验的企业事实、暂不能判断的内容。不要把专利排名直接表述为行业龙头排名。

## 第5轮：事实核验

把 `prompts/03_fact_check.md` 全文发送。

验收时追问：

> 请随机抽查正文20个数字，逐一展示它们在CSV结果表中的来源，不一致的立即修正。

## 第6轮：最终润色

把 `prompts/04_polish.md` 全文发送。

最后问：

> 请给出最终稿中最重要的5项结论，并说明每项结论的数据或权威来源。若证据不足，请降低表述强度。
