# 现有三份CSV对报告大纲的支持范围

| 报告内容 | 是否可做 | 主要数据 | 约束与修改建议 |
|---|---|---|---|
| 全国量子科技技术发展现状 | 不能仅靠CSV完成 | 权威网络资料 | 作为外部背景，优先使用政府、信通院、国家自然科学基金委、中科院及企业官方资料，所有动态事实注明来源和日期 |
| 上海量子专利规模及全国位势 | 可以 | 专利明细、企业年度表 | 当前单年只做截面；多年版本可做趋势、占比和排名变化 |
| 上海与北京、安徽、江苏、浙江、广东比较 | 可以，但需城市到省份映射 | 专利明细 | 当前`第一申请人地区`只有“中国”，比较依赖`第一申请人城市`映射；必须检查未映射城市 |
| 企业数量、企业专利和企业化程度 | 可以，需保守清洗 | 三表 | 申请人类型存在混合值，正文使用“明确识别为企业”口径，并附复核说明 |
| 企业梯队、头部集中度 | 可以 | 专利明细、企业汇总表 | 可依据专利数、核心相关度、活跃年份和集中度；不能称市场龙头或营收龙头 |
| 企业年度增长、新进入和持续创新 | 多年数据可做 | 企业年度表、专利明细 | 当前2021年测试版不可做；正式多年数据自动启用 |
| 技术方向结构 | 可以 | 专利明细中的`main_category` | 反映识别规则下的专利技术类别，不等同于完整技术路线评价 |
| 产业链环节结构 | 可以但需谨慎命名 | 专利明细中的`chain_position` | 这是基于关键词规则的技术/链条位置，不是完整投入产出产业链 |
| 专利质量 | 部分可以 | relevance、quantum_score、被引次数 | relevance和score反映识别相关度，不是法律或经济质量；当前被引覆盖率过低，不宜作为主指标 |
| 高校院所科研支撑 | 可以 | 申请人类型、专利明细 | 可分析专利数量、方向和主体；不能据此判断成果转化效率或合作关系 |
| 上海3家代表性企业 | 可以形成候选 | 企业排名、专利明细 | 企业的产品、经营和市场地位必须通过权威公开资料补充，不可仅凭专利推断 |
| 其他省市3—5家代表性企业 | 可以形成候选 | 全国企业排名 | 选择依据须明确为专利数据；避免声称覆盖所有优秀企业 |
| 产学研合作和合作网络 | 当前暂不分析 | 完整申请人字段可支持后续开发 | 本版按用户要求不做 |
| 营收、融资、就业、产值、市场份额 | 无法分析 | 需要工商、年报、投融资等外部数据 | 不得由专利数量推断 |
| 区级空间分布 | 无法可靠分析 | 需要地址或区县字段 | 当前仅城市层级 |
| 产品落地与应用场景 | CSV无法证明 | 企业官网、政府项目和权威新闻 | 必须逐条引用外部来源 |
